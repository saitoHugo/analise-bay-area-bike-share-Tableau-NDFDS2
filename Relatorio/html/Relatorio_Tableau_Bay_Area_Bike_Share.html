Dataset Bay Area Bike Share¶

Bay Area Bike Share uma empresa que oferece aluguel de bicicletas on-demand para clientes em San Francisco, Redwood City, Palo Alto, Mountain View e San Jose. Os usurios podem desbloquear bicicletas de uma variedade de estaes em cada cidade, e devolv-las em qualquer estao dentro da mesma cidade. Os usurios pagam o servio por meio de assinatura anual ou pela compra de passes de 3 dias ou 24 horas. Os usurios podem fazer um nmero ilimitado de viagens. Viagens com menos de trinta minutos de durao no tm custo adicional; Viagens mais longas incorrem em taxas de horas extras.

O anlise dos dados foi realizada em 4 etapas:

1) Qual o objetivo da anlise que est sendo realizada?

2) Limpeza dos Dados

3) Anlise Explortria

4) Visualizao no Tableau

Os dados coletados do site da empresa estavam divididos em 3 perodos de tempo ("201402", "201408", "201508"), onde para cada perodo existiam 3 arquivos:

  • Informaes sobre cada viagem ("..._trip_data.csv")
  • Informaes sobre as estaes ("..._station_data.csv")
  • Dados meteorolgicos dirios para cada cidade ("..._weather_data.csv")

#Definindo a funo que retorna o start_hour de maior movimento
def busy_hour(data_file):
    #empty dict with all possibles start_hour
    start_hours = {0:int(), 1:int(), 2:int(), 3:int(), 4:int(), 5:int(), 6:int(), 7:int(), 8:int(), 9:int(), 10:int(), 11:int(), 12:int(),
                   13:int(), 14:int(), 15:int(), 16:int(), 17:int(), 18:int(), 19:int(), 20:int(), 21:int(), 22:int(), 23:int()}
    #print type(start_hours.keys()[0])
    for hour in data_file['start_hour']:
        start_hours[hour] += 1
    return start_hours

Resumo¶

A histria criada evidencia os diferentes comportamentos dos usurios, onde os os usurios do tipo 'Subscriber' apresentam pico bem definidos nos horrios de rotina de trabalho(8h, 12h, 18h), j os usurios do tipo 'Customer' no apresentam picos de movimento, apenas ocorre um aumento (em uma taxa bem menor) entre o perodo das 9h as 18h. Em relao ao tempo mdio de uso constatou-se outra grande diferena, onde os usurios tipo 'Customer' realizam alugueis com grandes tempos de durao e os do tipo 'Subscriber' fazem o uso para pequenos deslocamentos. Quanto aos dias da semana de utilizao os usurios do tipo 'Subscriber' utilizam o servio predominantemente nos dias da semana. O oposto ocorre com os usurios do tipo 'customer'. Os usurios 'Subscriber' utilizam o servio como locomoo no dia a dia, em suas rotinas e no como lazer. O oposto ocorre com os do tipo 'Customer' onde ocorre um aumento de utilizao do servio aos finais de semana, porm a diferena de nmero de usurios nos dias de semana no to diferente aos finais de semana (quando compardo com os usurios do tipo 'subscriber') portanto esse tipo de usurios devem ser turistas visto que a regio um grande ponto turstico.

Design¶

Os dados que foram utilizados na visualizao inicial foram:

  • Tipo de usurio: dado categorico nominal
  • Durao da locao: dado quantitativo contnuo
  • Dia da semana: dado categrico ordinal (ordenado)
  • Hora de incio: varivel discreta

Inicialmente, escolhi explicar do que se tratava os dados e qual o objetivo com a anlise realizada. Aps deixar isso claro, achei interessantes comear mostrando uma das anlises exploratrias que realizei quando tive o primeiro contato com os dados. Mostrei a proporo entre cada tipo de usurio para um entendimento mais geral, e no menos importante, sobre os dados.

Aps essa introduo, iniciei a anlise explanatria dos dados. Mostrei o comportamento dos dois tipos de usurios para trs parmetros diferentes (hora de incio, durao e frequncia de utilizao ao longo dos dias da semana).

OBS: Cores para cada tipo de usurio facilitam a identificao e foram utilizadas azul para os usurios 'customer' e laranja para os usurios 'subscriber' para todos os grficos, criando um padro para todo o projeto.

1) Para mostrar os horrios de pico de utilizao do servio plotei um grfico de linha (eixo duplo) comparando os usurios. Mesmo nos dados a hora de incio ser classificado apenas como valores inteiros, utilizei o grfico de linha para enfatizar o padro geral dos usurios e plotei os dos usurios em um nico grfico, com diferentes cores, para facilitar a comparao.

2) Para evidenciar a diferena entre a mdia de durao entre os usurios, utilizei o grfico de baras por ser simples e objetivo para mostrar essa diferena. Cores auxiliam na comparao.

3) Visando comprar o comportamento dos dois usurios ao longo dos dias da semana, e como os dias da semana so dados categricos, escolhi novamente o grfico de barras, mas dessa vez preferi utilizar dois grficos, pois no queria comprar os valores (barra dupla para cada dia da semana em um nico grfico), gostaria de evidenciar a diferena de comportamento entre os dois usurios. Aqui, coloquei os ttulos dos grficos na cores padro de cada tipo de usurio para facilitar a associao do usurio e manter o pdro. Usei as mesmas cores para identificar os dias da semana nos dois grficos (nica legenda para os dois grficos) e inseri 'actions' para facilitar a comparao e melhorar a experincia de quem est visualizando a histria.

Aps evidenciar as diferenas encontradas, fui em busca de concluir para unir as infomaes encontradas. Com isso, criei uma nica visualizao com todas as 3 visualizaes explanatrias e criei iteratividade entre elas visando destadar todos os dados de cada tipo de usurios, bem como comprar os dias da semana. Por fim, coloquei um texto conclusivo para o fechamento da ideia encontrada. Caracterizei os dois diferentes tipo de usurios e seus custumes com base nos dados do aluguel das bicicletas.
